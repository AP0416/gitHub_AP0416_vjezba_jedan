# gitHub_AP0416_vjezba_jedan
Upoznavanje s osnovama gita

## 1. Što je Visual Studio Code i za što se najčešće koristi?
Visual Studio Code je uređivač izvornog koda, najčesšće se koristi za uređivanje koda.
## 2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
File explorer
## 3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
Panel za ekstenzije onda plugin i odaberemo ekstenziju i skinemo ju. Ekstenzije služe za dodavanje novih jezika i alata.
## 4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?
File onda open file i nađemo datoteku
## 5. Što se sve može raditi u Exploreru?
Možemo pregledavati datoteke.
## 6. Kako možeš brzo pronaći određenu datoteku u projektu?
Crtl+P
## 7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?
Ctrl+Shift+P i Split Editor
## 8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
Crtl+Shift+P i onda Format Document
## 9. Što je version control i zašto se koristi?
Korisiti se kako bi se mogli vratiti na staru verziju
## 10. Kako se u VS Code-u otvara panel za Git (Source Control)?đ
Ctrl+Shift+G ili kliknemo sa strane
## 11. Što znače commit, push, pull?
commit - mijenja verziju koda na računalu
push - mijenja verziju koda na serveru
pull - preuzima promjene sa servera i stavlja ih na računalo
# 12. Čemu služi commit poruka i kako treba izgledati?
Služi da znamo što je promjenjeno i zašto. Treba biti kratka i jasna
## 13. Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
Možemo gledati Version Control Panel u Visual Studio Code-u ili na GitHub-u starije verzije
## 14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
Commitamo lokalnu verziju i onda push na GitHub
## 15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
Možemo koristiti git reset
## 16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
Možemo koristiti git status
